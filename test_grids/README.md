# Test Grids

## Public Grids (distributed to participants)

Each grid type is provided at four sizes: 512, 2048, 8192, and 32768.
Files are named `<type>_<size>.bin`, e.g. `public_1_random_low_512.bin`.

| Type | Description | ~Adult % |
|---|---|---|
| `public_1_random_low` | Random, ~5% adults | 5% |
| `public_2_random_high` | Random, ~50% adults | 50% |
| `public_3_structured` | Repeating 3×3 blocks on 10-cell pitch | 9% |
| `public_4_sparse_clusters` | Dense circular clusters in empty field | varies |
| `public_5_boundary_stress` | Dense fill along all four toroidal edges | varies |

Each `.bin` has a matching `.expected.bin` — the generation-10,000 output
produced by the reference implementation.

| Size | File size |
|---|---|
| 512 × 512 | ~0.3 MiB |
| 2048 × 2048 | ~4 MiB |
| 8192 × 8192 | ~64 MiB |
| 32768 × 32768 | ~1024 MiB |

## Generating Grids

Initial grids (inputs) are generated by `generate.py`:

```bash
sudo apt-get install python3-numpy   # or: pip install numpy
python3 test_grids/generate.py --output-dir test_grids
```

## Generating Expected Outputs

Expected outputs require running the reference implementation on each input.
Small grids (512, 2048) complete in seconds to minutes. Large grids (8192,
32768) take hours — run them in a screen session on the c8g.2xlarge.

```bash
bash reference/build.sh

for name in public_1_random_low public_2_random_high public_3_structured \
            public_4_sparse_clusters public_5_boundary_stress; do
    for size in 512 2048 8192 32768; do
        echo "Simulating ${name}_${size} ..."
        ./spawn_sim test_grids/${name}_${size}.bin \
                    test_grids/${name}_${size}.expected.bin
    done
done
```

The 32768 grids take ~25 hours each on the reference implementation.
Run at most two concurrently (RAM budget: 2 × 2 GiB grid buffers on 16 GiB).

Commit the `.expected.bin` files to the repo. Use Git LFS for the 32768 ones
(~1 GiB each); the smaller sizes can be committed directly.
